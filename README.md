# pizzeria_database

Учебный проект по проектированию и анализу реляционной базы данных локальной пиццерии (г. Нижний Новгород).

## Цели проекта
- Проектирование логически корректной структуры БД;
- Использование SQL-представлений для аналитических задач;
- Оптимизация запросов с помощью индексов;
- Анализ и визуализация данных на основе SQL views.

## Стек
- PostgreSQL
- SQL (DDL, DML, Views, Indexes)
- Python (pandas, matplotlib, Faker)
- Power BI

## Структура БД
ER-диаграмма во вложении отражает отношения сущностей друг к другу.

## Индексы
- `idx_orders_date_time` 
- `idx_orders_customer_date`
- `idx_order_items_pizza` 
- `idx_customer_reg_date` 

## Представления
- `last_day_sales` — суммарные дневные продажи
- `new_customers_per_month` — количество новых клиентов за месяц
- `total_orders` — самые популярные пиццы
- `delivery_efficiency` — эффективность курьеров
- `unactive_clients` — клиенты без заказов за последний год 

## Аналитика
Данные агрегированы в SQL\Pandas.
Визуализация построена в Power BI на основе данных из указанных представлений.

Примеры:
- Динамика выручки по дням
- Топ-товары по сумме продаж
- Выручка по группам товаров
