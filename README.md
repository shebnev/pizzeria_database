# pizzeria_database

Проектирование и анализ реляционной базы данных локальной пиццерии (г. Нижний Новгород).

## Цели проекта

* Проектирование логически корректной структуры БД;
* Использование SQL-представлений для аналитических задач;
* Оптимизация запросов;
* Анализ и визуализация данных.

## Стек

* **PostgreSQL** (DDL, DML, Views, Indexes)
* **Python** (pandas, Faker)
* **Power BI**

## Структура БД

ER-диаграмма отражает основные сущности, их связи и ключевые ограничения.

## Индексы

Оптимизация типовых аналитических и фильтрующих запросов:

* `idx_orders_date_time`
* `idx_orders_customer_date`
* `idx_order_items_pizza`
* `idx_customer_reg_date`

## Представления

Для аналитики используются SQL-представления:

* `last_day_sales` — агрегированные дневные продажи;
* `new_customers_per_month` — количество новых клиентов за месяц;
* `total_orders` — популярность товаров по количеству заказов;
* `delivery_efficiency` — показатель эффективности курьеров;
* `unactive_clients` — клиенты без заказов за последний год.

## Аналитика

Агрегации выполняются на уровне SQL и Pandas.
Визуализация построена в Power BI.

Примеры аналитики:

* динамика выручки по дням;
* топ товары по количеству продаж;
* выручка по типам товаров.
